## Phase 1：把“制作视频”流转合同定死（数据字段 + 路由入口）
### 1.1 统一入口路由
- 以现有路由 `GET /festival/category/video` 作为视频选择页（已存在并指向 `VideoCategoryPage`）。
- 为兼容当前首页硬编码与文档描述，**增加一个别名路由**：`/festival/video-category` → 同一个 `VideoCategoryPage`。
  - 目标：
    - 旧入口不 404（HomePageGlass 目前写死 `/festival/video-category`）
    - 新入口统一到 `/festival/category/video`

### 1.2 统一携带字段（只走 NavigationState）
- 所有“制作视频”触发点（ResultPage / ContinueCreationPanel / MaterialLibraryPage / TextPage / VoicePageNew 等）都改为：
  1) `const navState = createNavigationState({ image, text, audio, ...source })`
  2) `navigate('/festival/category/video', { state: navState })`
- 禁止再出现“散装字段”如 `{ image, audioUrl, caption }`（当前素材库组合就是这种），避免 VideoPage/KlingEffectsPage 读不到数据。

### 1.3 缺参降级规则（写进页面逻辑 + 文案）
- **数字人拜年**：至少需要 image；缺 image → 禁止进入生成态，提供“去素材库选图 / 去生成头像 / 去上传”。
- **Kling 特效**：至少需要 image；缺 image → 同上。
- 仅 text：允许进入选择页，但两个选项都提示“缺图片”，并给出补齐路径。
